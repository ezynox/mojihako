原稿用紙とは

原稿用紙は、一定のマス目に文字を一字ずつ書き込む用紙です。 縦書き原稿用紙では文字が上から下へ並び、列は右から左へ進みます。

200字原稿用紙と400字原稿用紙の違い

400字詰め原稿用紙は20字×20行（20×20）が基準です。 200字詰め原稿用紙は20字×10行（20×10）が基準です。

縦書き原稿用紙の基本ルール

1マスに1文字を配置します。改行すると次の列に移ります。 「段落の最初を一マス空ける」オプションをオンにすると、段落の先頭に字下げが入ります。 句読点や括弧類も、原則として1マスを使います。

単語間にスペースを使う文章（英文など）では、行頭に来るスペースはスキップされ、 先頭のマスが空白で始まらないように処理します。

「禁則処理」をオンにすると、「。」「、」「」」などの句読点・閉じ括弧が 行頭に来ないよう、直前の行末に添わせて表示します。

文字数基準の枚数と実際の配置基準の枚数の違い

「文字数換算」は総文字数を原稿用紙のマス数で割った目安です。 「原稿用紙換算」は改行・空行・字下げ・禁則処理など、 実際にマスを使う要素を反映した目安です。

TXT ファイルの読み込みについて

ファイルの読み込みにはブラウザの File API を使用しており、 入力テキストもファイルの内容もサーバーには送信されません。

注意事項

原稿用紙の書き方ルールは、学校・公募・出版社・提出先によって異なります。 このツールは縦書きレイアウトの確認を補助する目的で作られており、 専門的な組版・出版編集の代わりにはなりません。 実際の提出用原稿を作成する際は、各提出先のルールを必ずご確認ください。

縦書き原稿用紙のGuide sectionと枚数計算ガイド

作文の課題やエッセイ、公募原稿、翻訳文を仕上げるとき、 テキストが実際の原稿用紙にどう収まるかを確認したい場面があります。 縦書き原稿用紙は文字を上から下へ、行を右から左へ並べる日本語の文章でよく使われる書式で、 字下げ・改行・句読点の位置といったルールが絡むため、 単純な文字数だけでは枚数を正確に見積もりにくいことがあります。

このビューアはテキストを縦書き原稿用紙形式に配置し、 総文字数・空白除き文字数・文字数換算の枚数・実際の配置に基づく枚数をまとめて計算します。 400字と200字の切り替えも自由にできます。

1. 縦書き原稿用紙とは

原稿用紙は一定のマス目に文字を書き込む用紙で、1マスに1文字を入れるのが基本です。 縦書きでは文字が上から下へ、列が右から左へ進みます。

標準的な400字詰め原稿用紙は20字×20行（20×20）。 200字詰めは20字×10行（20×10）です。 課題や公募で「原稿用紙○枚以内」と指定されている場合は、 どちらの形式かを必ず確認しましょう。

2. 文字数で計算した枚数と実際の枚数がずれる理由

文字数を400や200で割るだけでは、実際の配置と差が出ることがあります。 原稿用紙では、段落の字下げ・改行後の余白・空行・禁則処理といった要素が マスを消費するためです。

  • 文字数換算：総文字数 ÷ マス数
  • 原稿用紙換算：改行・空行・字下げ・禁則処理を反映した値

たとえば文字数が400字未満でも、段落が多かったり改行が多かったりすると、 実際の原稿用紙では2枚目にはみ出すことがあります。 提出枚数を合わせたいときは両方の数値を確認するのが安全です。

3. 縦書き原稿用紙の基本ルール

各オプションを切り替えると、レイアウトへの影響をリアルタイムで確認できます。

  • 字下げ：段落の先頭マスを1字分空けます。
  • 改行を反映：入力テキストの改行を配置に反映します。
  • 空行を削除：空の行を削除して配置に反映しないようにできます。
  • 禁則処理：句読点や閉じ括弧が列の先頭に来ないよう調整します。

「。」「、」「」」などが行頭に来ると、縦書きでは読みにくくなることがあります。 禁則処理をオンにすると、これらの文字をなるべく直前の列末尾に添わせて 原稿用紙らしい仕上がりに近づけます。

4. 400字詰めと200字詰めの違い

400字詰め原稿用紙は作文・文学作品・課題・公募でよく使われる標準形式です。 1ページに多くの文字が入るので、長い文章の分量確認に向いています。

200字詰めは1ページの列数が少なく、短い文章や練習用の確認に適しています。 同じテキストでも200字基準では400字基準より枚数が多く表示されます。

5. TXT・MD ファイルの読み込みとプライバシー

メモ帳・Markdown エディタ・ワープロなどで書いたテキストを .txt または .md ファイルとして読み込めます。 長い文章を毎回コピー&ペーストする手間が省けます。

ファイルの読み込みにはブラウザの File API を使用しています。 入力したテキストもファイルの内容も外部サーバーには送信されず、 すべての処理はブラウザ内で完結します。

よくある質問

Q. 日本語以外のテキストも配置できますか？

はい。漢字・ひらがな・カタカナのほか、英字・数字・記号が混在したテキストも配置できます。

全角アルファベット・数字（Ａ、ａ、１ など）は 回転せず1文字1マスとして扱います。 半角アルファベット・数字は縦書きグリッド内で回転表示され、 1マスに最大2文字まで配置されます。

Q. 空白や改行も文字数に含まれますか？

全文字数と空白除きの文字数をそれぞれ表示します。 配置基準の枚数では、改行・空行・字下げなど マスを消費する要素も反映されます。

Q. 印刷したり、提出用ファイルとして保存できますか？

このツールはブラウザ内でのレイアウト確認と枚数計算を目的としています。 実際の提出用原稿用紙が必要な場合は、 提出先から指定された用紙や書式をご利用ください。
